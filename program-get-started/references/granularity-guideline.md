## 粒度划分使用指南

### 粒度选择原则
| 学习场景 | 推荐粒度 | 说明 |
|----------|----------|------|
| 零基础入门 | 3级（阶段 → 模块 → 知识点） | 重点是建立整体认知，避免陷入细节 |
| 有基础提升 | 4级（阶段 → 模块 → 章节 → 知识点） | 系统梳理知识体系，查漏补缺 |
| 面试突击/深入研究 | 5级（阶段 → 模块 → 章节 → 知识点 → 子知识点） | 精准掌握细节，应对深度考察 |

### 拆分原则
- **适度拆分**：每个子知识点应该是一个独立的、可学习的单元，避免拆分过细导致碎片化
- **逻辑连贯**：同一章节下的知识点应该具有强相关性
- **难度递进**：从易到难，从基础到高级，符合学习曲线
- **避免重复**：相同内容不要在多个地方重复出现

### 任务分配建议
- **简单知识点**：单个子agent负责完整知识点（30-60分钟）
- **中等复杂度知识点**：拆分2-3个子任务，由不同子agent负责（每个20-30分钟）
- **复杂知识点**：按子知识点拆分多个任务，并行执行（每个10-20分钟）

## 示例：集合框架五级结构

```
阶段：Java 高级特性
├── 模块：集合框架
│   ├── 章节：List 接口
│   │   ├── 知识点：ArrayList
│   │   │   ├── 子知识点：底层数据结构
│   │   │   ├── 子知识点：扩容机制
│   │   │   └── 子知识点：常用方法时间复杂度
│   │   └── 知识点：LinkedList
│   │       ├── 子知识点：双向链表实现
│   │       ├── 子知识点：与ArrayList性能对比
│   │       └── 子知识点：适用场景
│   ├── 章节：Map 接口
│   │   ├── 知识点：HashMap
│   │   │   ├── 子知识点：哈希表原理
│   │   │   ├── 子知识点：哈希冲突解决
│   │   │   ├── 子知识点：JDK7与JDK8实现差异
│   │   │   └── 子知识点：扩容机制
│   │   └── 知识点：ConcurrentHashMap
│   │       ├── 子知识点：分段锁原理（JDK7）
│   │       ├── 子知识点：CAS+synchronized实现（JDK8）
│   │       └── 子知识点：并发性能分析

│       ├── 知识点：HashSet

```

## 适配说明

### 对于现有计划的迁移
1. 原有两级结构（阶段→知识点）可以平滑升级到五级结构
2. 不需要强制所有内容都使用五级，保持灵活性
3. 优先在核心模块（集合、多线程、JVM、Spring）中使用五级结构

### 内容生成效率
- 五级结构不会显著增加生成时间，因为可以并行生成子知识点
- 细粒度拆分更适合多agent并行执行，提高整体生成效率
- 便于局部更新和修正，不需要重新生成整个模块的内容

### 学习体验优化
- 学习者可以根据自身情况选择学习深度
- 进度追踪更精准，可以精确到子知识点级别
- 便于制定个性化学习计划，跳过已掌握的内容
