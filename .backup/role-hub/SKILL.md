# role-hub - AI智能团队角色切换

自动切换到指定的专业角色,提供对应领域的专业支持。

## ⚡ 执行方式

检测用户输入中的角色标识符(@PM、@UI、@ARCH、@DEV、@QA),自动加载对应角色定义。

### 角色识别

当用户输入包含以下标识时,自动激活对应角色:

- **@PM** - 产品经理
- **@UI** - UI设计师
- **@ARCH** - 架构师
- **@DEV** - 开发工程师
- **@QA** - 质量工程师

### 执行流程

```bash
python3 ~/.claude/skills/role-hub/role_switcher.py "用户的完整输入"
```

脚本会:
1. 识别输入中的角色标识(@PM/@UI/@ARCH/@DEV/@QA)
2. 从角色库加载对应的角色定义
3. 返回角色的完整上下文(职责、技能、输出物等)
4. Claude 将以该角色的视角和风格回答问题

## 核心特性

- ✅ **自动识别** - 检测 @角色代号 自动切换
- ✅ **完整上下文** - 加载角色的全部定义信息
- ✅ **专业视角** - 以角色的专业能力和风格回答
- ✅ **协作流程** - 支持加载协作流程文档
- ✅ **灵活扩展** - 支持自定义角色和流程
- ✅ **文档交接** - 每个角色完成后产出标准化交接文档
- ✅ **任务清单** - 开发角色自动管理任务清单,实时更新进度
- ✅ **前置检查** - 自动检查角色执行的前置条件
- ✅ **断点续做** - 任务上下文持久化,防止中断

## 使用示例

### 单角色调用
```
用户: @PM 我们想做一个用户积分系统
→ 激活产品经理角色,从产品视角分析需求

用户: @ARCH 设计一个高并发秒杀系统
→ 激活架构师角色,提供技术架构方案

用户: @DEV 实现用户注册功能
→ 激活开发工程师角色,提供代码实现
```

### 流程调用
```
用户: 启动需求评审流程
→ 加载需求评审流程文档,指导多角色协作

用户: 开始技术设计
→ 加载技术设计流程文档
```

## 配置说明

角色定义文件位于:
- **全局角色库**: `~/.claude/skills/role-hub/roles/`
- **项目角色库**: `<项目根目录>/role-hub/roles/`

优先级: 项目角色库 > 全局角色库

## 角色工作流程

每个角色都遵循标准的工作流程:

### 1. 开始任务前
- ✅ 检查前置条件 (是否有上游角色的产出物)
- ✅ 理解需求和上下文
- ✅ 生成工作文档模板

### 2. 执行任务中
- ✅ 按照角色职责完成工作
- ✅ 实时更新任务进度 (开发角色)
- ✅ 记录关键决策和问题

### 3. 完成任务后
- ✅ 产出标准化交接文档
- ✅ 明确列出交接清单
- ✅ 保存任务上下文

## 协作机制

### 前置条件依赖
- **UI设计** 需要先有 **PRD**
- **架构设计** 需要先有 **PRD**
- **开发** 需要先有 **技术设计文档**,建议有 **UI设计稿**
- **测试** 需要先有 **开发交付文档**

### 产出物交接
每个角色完成后会产出标准文档:
- **PM** → PRD文档 → 交接给 UI/ARCH
- **UI** → UI设计文档 → 交接给 DEV
- **ARCH** → 技术设计文档 → 交接给 DEV
- **DEV** → 开发清单 + 交付文档 → 交接给 QA
- **QA** → 测试报告 → 交接给 PM/DEV

所有产出物保存在: `.role-hub/artifacts/<任务名>/`

## 文档模板

系统提供标准化文档模板:
- **PM**: PRD文档模板 (`pm-prd.md`)
- **UI**: UI设计交付文档模板 (`ui-design-spec.md`)
- **ARCH**: 技术设计文档模板 (`arch-design.md`)
- **DEV**: 开发任务清单模板 (`dev-checklist.md`)
- **QA**: 测试报告模板 (`qa-test-report.md`)

模板位置: `~/.claude/skills/role-hub/templates/`

## 工具命令

### 文档生成
```bash
python3 ~/.claude/skills/role-hub/doc_generator.py generate <角色> <任务名>
```

### 上下文管理
```bash
# 保存上下文
python3 ~/.claude/skills/role-hub/context_manager.py save <任务名> <角色> <数据>

# 加载上下文
python3 ~/.claude/skills/role-hub/context_manager.py load <任务名>

# 检查前置条件
python3 ~/.claude/skills/role-hub/context_manager.py check <角色> <任务名>

# 查看产出物
python3 ~/.claude/skills/role-hub/context_manager.py artifacts <任务名>
```

## 角色切换规则

1. 识别到 @角色代号 后立即切换
2. 自动检查前置条件是否满足
3. 保持角色视角直到下次切换
4. 回答风格符合角色定位
5. 完成后产出标准化交接文档
6. 首先声明当前角色身份
