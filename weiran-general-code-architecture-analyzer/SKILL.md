# 代码架构分析器技能

## 核心功能
本技能用于系统化分析项目代码，输出完整的架构评估报告，包含以下内容：
1. **项目架构分析**：技术栈识别、目录结构梳理、模块划分、依赖关系
2. **业务逻辑梳理**：核心流程、关键功能模块、调用链路分析
3. **冗余问题检测**：重复代码、未使用的文件/函数/依赖、过时逻辑
4. **设计缺陷识别**：耦合度过高、职责划分不清、扩展性不足、性能瓶颈
5. **遗漏点分析**：缺失的错误处理、安全校验、日志记录、测试覆盖
6. **优化建议**：按优先级排序的改进方案和重构建议

## 执行步骤
### 1. 项目扫描阶段
- 使用`Glob`工具扫描项目根目录，识别核心目录结构
- 使用`Grep`工具搜索配置文件，识别技术栈、框架、依赖版本
- 读取README、package.json、composer.json等配置文件了解项目基本信息

### 2. 架构分析阶段
- 梳理项目分层结构（MVC/DDD/微服务等）
- 识别核心模块及职责划分
- 分析模块间依赖关系和调用链路
- 评估架构的可扩展性、可维护性

### 3. 业务逻辑分析阶段
- 定位核心业务功能入口（控制器、路由、命令行等）
- 梳理关键业务流程的执行路径
- 识别核心领域模型和业务规则
- 分析业务逻辑的复杂度和合理性

### 4. 问题检测阶段
#### 冗余检测：
- 查找重复代码块（相似度 > 70%）
- 识别未使用的文件、函数、类、依赖包
- 检测过时的API调用和废弃的逻辑
- 查找冗余的配置和资源文件

#### 设计缺陷检测：
- 识别强耦合的模块和类
- 检测职责不单一的函数/类
- 评估性能瓶颈点（N+1查询、循环调用等）
- 分析错误处理和异常捕获的完整性

#### 遗漏点识别：
- 缺失的参数校验和安全防护
- 关键路径缺少日志记录
- 未处理的异常和边缘情况
- 缺失的单元测试和集成测试
- 未实现的功能点和业务规则

## 使用说明
- 支持任意规模的项目分析，对于大型项目会自动分层级扫描
- 分析过程中会自动跳过node_modules、vendor、.git等第三方目录
- 所有问题都会标注对应的文件路径和行号，方便定位
- 优化建议会考虑项目实际情况，避免过度设计
