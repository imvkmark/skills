# Go 项目分析要点

针对 Go 项目（Gin、Echo、gRPC 等）的深度分析指导。

## 项目类型检测

**特征文件**:
- `go.mod`
- `go.sum`

**关键依赖识别**:
- Gin / Echo / Fiber (Web 框架)
- GORM / sqlc (ORM/SQL)
- gRPC (RPC 框架)
- Cobra (CLI 框架)
- testify (测试框架)

## 架构模式识别

### 单体架构特征
- 单一的 `go.mod`
- 统一的项目目录结构
- 共享的数据库配置

### 微服务架构特征
- 多个独立的服务目录
- 每个服务有独立的 `main.go`
- 服务间通过 gRPC/HTTP 通信

## 核心分析要点

### 1. 处理器层分析

**文件位置**: `internal/handler/` 或 `pkg/handler/`

**分析重点**:
- 路由定义
- 请求处理逻辑
- 响应格式
- 错误处理
- 中间件使用

**检查清单**:
- [ ] 处理器是否遵循单一职责原则
- [ ] 是否存在过度的业务逻辑处理
- [ ] 参数验证是否充分
- [ ] 错误响应格式是否统一
