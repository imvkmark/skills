# PHP/Weiran 项目分析要点

针对 PHP 项目（特别是 Weiran 框架 Laravel 10.x + PHP 8.2）的深度分析指导。

## 项目类型检测

**特征文件**:
- `composer.json`
- `composer.lock`

**关键依赖识别**:
- Laravel (Web 框架)
- Weiran (模块化框架)
- Eloquent (ORM)
- Laravel Queue (任务队列)
- PHPUnit / Pest (测试框架)

## 架构模式识别

### 单体架构特征
- 单一的 `composer.json`
- 统一的项目目录结构
- 共享的数据库配置

### 模块化单体架构（Weiran 特性）
- `modules/` 目录下的多个模块
- 每个模块有独立的配置和资源
- 模块间通过事件系统通信

## 核心分析要点

### 1. 控制器层分析

**文件位置**: `modules/*/src/Http/*/` 或 `app/Http/Controllers/`

**分析重点**:
- 路由定义和控制器映射
- 请求验证 (FormRequest)
- 响应格式和资源转换
- 中间件使用
- 异常处理

**检查清单**:
- [ ] 控制器是否遵循单一职责原则
- [ ] 是否存在过度的业务逻辑处理
- [ ] FormRequest 验证是否充分
