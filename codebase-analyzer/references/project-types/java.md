# Java 项目分析要点

针对 Java 项目（Spring Boot、Spring Cloud、Quarkus 等）的深度分析指导。

## 项目类型检测

**特征文件**:
- `pom.xml` (Maven)
- `build.gradle` (Gradle)
- `settings.gradle`

**关键依赖识别**:
- Spring Framework / Spring Boot
- Spring Cloud (微服务)
- Hibernate / JPA (ORM)
- MyBatis (SQL Mapper)
- Dubbo / gRPC (RPC)

## 架构模式识别

### 单体架构特征
- 单一 `pom.xml` 或 `build.gradle`
- 统一的 `src/main/java` 目录结构
- 共享的数据库配置

### 微服务架构特征
- 多个独立的模块，每个有自己的 `pom.xml`
- 每个服务有独立的启动类 (`@SpringBootApplication`)
- 服务间通过 HTTP/gRPC 通信

## 核心分析要点

### 1. 控制器层分析

**文件位置**: `src/main/java/**/controller/` 或 `src/main/java/**/web/`

**分析重点**:
- 路由映射 (`@RequestMapping`, `@GetMapping`, `@PostMapping` 等)
- 请求参数处理 (`@RequestParam`, `@PathVariable`, `@RequestBody`)
- 响应格式 (`@ResponseBody`, `ResponseEntity`)
- 异常处理 (`@ExceptionHandler`)
- 中间件和拦截器的应用

**检查清单**:
- [ ] 控制器是否遵循单一职责原则
- [ ] 是否存在过度的业务逻辑处理
- [ ] 参数验证是否充分 (`@Valid`, `@Validated`)
- [ ] 错误响应格式是否统一
- [ ] 是否正确使用了 HTTP 状态码

### 2. 服务层分析

**文件位置**: `src/main/java/**/service/`

**分析重点**:
- 服务接口定义
- 业务逻辑实现
- 事务管理 (`@Transactional`)
- 依赖注入 (`@Autowired`, `@Inject`)
- 缓存使用 (`@Cacheable`, `@CacheEvict`)

**检查清单**:
- [ ] 服务是否正确封装业务逻辑
- [ ] 事务边界是否合理
- [ ] 是否存在循环依赖
- [ ] 缓存策略是否合理
- [ ] 是否有适当的日志记录

### 3. 数据访问层分析

**文件位置**: `src/main/java/**/repository/` 或 `src/main/java/**/dao/`

**分析重点**:
- ORM 框架使用 (Hibernate, JPA, MyBatis)
- 数据库连接池配置
- SQL 查询优化
- N+1 查询问题
- 事务隔离级别

**检查清单**:
- [ ] 是否存在 N+1 查询问题
- [ ] 是否使用了适当的查询优化 (join fetch, batch)
- [ ] 连接池配置是否合理
- [ ] 是否有适当的索引
- [ ] 是否存在 SQL 注入风险

### 4. 模型层分析

**文件位置**: `src/main/java/**/model/` 或 `src/main/java/**/entity/`

**分析重点**:
- 实体关系映射 (`@OneToOne`, `@OneToMany`, `@ManyToMany`)
- 继承策略 (`@Inheritance`)
- 字段映射 (`@Column`, `@Transient`)
- 验证注解 (`@NotNull`, `@Size` 等)

**检查清单**:
- [ ] 关联关系是否正确定义
- [ ] 是否存在循环引用
- [ ] 是否使用了懒加载/急加载策略
- [ ] 字段验证是否充分
- [ ] 是否有适当的 getter/setter

### 5. 配置管理分析

**文件位置**: `src/main/resources/application.yml` 或 `application.properties`

**分析重点**:
- 数据库配置
- 缓存配置
- 日志配置
- 自定义配置属性
- 环境特定配置

**检查清单**:
- [ ] 是否使用了配置中心 (Nacos, Consul)
- [ ] 敏感信息是否被妥善保护
- [ ] 配置是否支持多环境
- [ ] 是否有合理的默认值

### 6. 依赖注入分析

**分析重点**:
- Bean 定义 (`@Component`, `@Service`, `@Repository`)
- 依赖注入方式 (构造器、setter、字段)
- 循环依赖检测
- Bean 生命周期管理

**检查清单**:
- [ ] 是否存在循环依赖
- [ ] 是否使用了构造器注入 (推荐)
- [ ] Bean 作用域是否合理 (singleton, prototype)
- [ ] 是否有不必要的 Bean 创建

### 7. 异常处理分析

**分析重点**:
- 自定义异常定义
- 异常处理策略
- 全局异常处理器
- 日志记录

**检查清单**:
- [ ] 是否有统一的异常处理机制
- [ ] 异常信息是否清晰
- [ ] 是否正确处理了检查异常和非检查异常
- [ ] 是否有适当的错误日志

### 8. 并发与线程安全

**分析重点**:
- 线程池配置
- 同步机制使用
- 并发集合使用
- 原子操作

**检查清单**:
- [ ] 是否存在竞态条件
- [ ] 是否正确使用了同步机制
- [ ] 线程池配置是否合理
- [ ] 是否有死锁风险

### 9. 性能优化

**分析重点**:
- 缓存策略
- 数据库查询优化
- 连接池配置
- 异步处理

**检查清单**:
- [ ] 是否有缓存策略
- [ ] 是否存在 N+1 查询
- [ ] 连接池是否配置合理
- [ ] 是否使用了异步处理 (CompletableFuture, @Async)

### 10. 安全性分析

**分析重点**:
- 认证机制 (Spring Security)
- 授权控制
- 输入验证
- SQL 注入防护
- XSS 防护

**检查清单**:
- [ ] 是否有认证机制
- [ ] 是否有适当的授权控制
- [ ] 是否验证了所有用户输入
- [ ] 是否使用了参数化查询
- [ ] 是否有 CSRF 防护

### 11. 测试覆盖

**文件位置**: `src/test/java/`

**分析重点**:
- 单元测试覆盖率
- 集成测试
- Mock 框架使用 (Mockito, PowerMock)
- 测试数据准备

**检查清单**:
- [ ] 单元测试覆盖率是否足够
- [ ] 是否有集成测试
- [ ] 测试是否独立且可重复
- [ ] 是否使用了测试容器 (Testcontainers)

### 12. 日志与监控

**分析重点**:
- 日志框架 (SLF4J, Log4j)
- 日志级别使用
- 结构化日志
- 监控指标 (Micrometer, Prometheus)

**检查清单**:
- [ ] 是否有适当的日志记录
- [ ] 日志级别是否合理
- [ ] 是否有性能监控
- [ ] 是否有健康检查端点

## 微服务特定分析要点

### 服务通信

**分析重点**:
- REST API 设计
- gRPC 接口定义
- 消息队列使用 (RabbitMQ, Kafka)
- 服务调用超时和重试

**检查清单**:
- [ ] API 是否遵循 RESTful 设计
- [ ] 是否有服务降级和熔断机制
- [ ] 是否有适当的超时设置
- [ ] 是否有重试机制

- 限流和熔断 (Hystrix, Resilience4j)

**检查清单**:
- [ ] 是否使用了服务注册中心
- [ ] 是否有负载均衡策略
- [ ] 是否有熔断机制
- [ ] 是否有分布式追踪

### 数据一致性

**分析重点**:
- 分布式事务处理
- 最终一致性设计
- 事件驱动架构

**检查清单**:
- [ ] 如何处理分布式事务
- [ ] 是否使用了 Saga 模式
- [ ] 是否有事件驱动设计
- [ ] 数据一致性如何保证

- [ ] 安全风险都被评估了
