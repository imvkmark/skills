# Node.js 项目分析要点

针对 Node.js 项目（Express、NestJS、Fastify 等）的深度分析指导。

## 项目类型检测

**特征文件**:
- `package.json`
- `package-lock.json` 或 `yarn.lock`

**关键依赖识别**:
- Express / NestJS / Fastify (Web 框架)
- Sequelize / TypeORM / Prisma (ORM)
- Bull / BullMQ (任务队列)
- Jest / Mocha (测试框架)
- TypeScript (类型系统)

## 架构模式识别

### 单体架构特征
- 单一的 `package.json`
- 统一的项目目录结构
- 共享的数据库配置

### 微服务架构特征
- 多个独立的服务目录
- 每个服务有独立的 `package.json`
- 服务间通过 HTTP/gRPC 通信

## 核心分析要点

### 1. 路由层分析

**文件位置**: `src/routes/` 或 `src/controllers/`

**分析重点**:
- 路由定义
- 请求处理逻辑
- 响应格式
- 错误处理
- 中间件使用

**检查清单**:

- [ ] 参数验证是否充分
