# Python 项目分析要点

针对 Python 项目（Django、Flask、FastAPI、Celery 等）的深度分析指导。

## 项目类型检测

**特征文件**:
- `requirements.txt`
- `setup.py`
- `pyproject.toml`
- `Pipfile`
- `poetry.lock`

**关键依赖识别**:
- Django / Flask / FastAPI (Web 框架)
- SQLAlchemy / Django ORM (ORM)
- Celery / RQ (任务队列)
- Pydantic (数据验证)
- pytest / unittest (测试框架)

## 架构模式识别

### 单体架构特征
- 单一的 `requirements.txt`
- 统一的项目目录结构
- 共享的数据库配置

### 微服务架构特征
- 多个独立的服务目录
- 每个服务有独立的 `requirements.txt`
- 服务间通过 HTTP/gRPC 通信

## 核心分析要点

### 1. 视图层分析

**文件位置**: `app/views.py` 或 `app/routes.py` 或 `app/api/`

**分析重点**:
- 路由定义 (`@app.route()`, `@router.get()` 等)
- 请求处理逻辑
- 响应格式
- 错误处理
- 中间件使用

**检查清单**:
- [ ] 视图函数是否遵循单一职责原则
- [ ] 是否存在过度的业务逻辑处理
- [ ] 参数验证是否充分
- [ ] 错误响应格式是否统一
- [ ] 是否正确使用了 HTTP 状态码

### 2. 业务逻辑层分析

**文件位置**: `app/services/` 或 `app/business/`

**分析重点**:
- 业务逻辑实现
- 依赖注入模式
- 异常处理
- 缓存使用
- 日志记录

**检查清单**:
- [ ] 业务逻辑是否正确封装
- [ ] 是否存在循环依赖
- [ ] 异常处理是否合理
- [ ] 缓存策略是否合理
- [ ] 是否有适当的日志记录

### 3. 数据访问层分析

**文件位置**: `app/models.py` 或 `app/repositories/`

**分析重点**:
- ORM 使用 (SQLAlchemy, Django ORM)
- 数据库连接管理
- 查询优化
- N+1 查询问题
- 事务管理

**检查清单**:
- [ ] 是否存在 N+1 查询问题
- [ ] 是否使用了查询优化 (select_related, prefetch_related)
- [ ] 连接池配置是否合理
- [ ] 是否有适当的索引
- [ ] 是否存在 SQL 注入风险

### 4. 模型层分析

**文件位置**: `app/models.py`

**分析重点**:
- 模型定义
- 字段类型和约束
- 关系定义 (ForeignKey, ManyToMany)
