# Project Analyzer 技能

通用项目代码分析工具，支持多种编程语言和架构模式。

## 功能特性

✅ **多语言支持**
- Java (Spring Boot, Spring Cloud, Quarkus)
- Python (Django, Flask, FastAPI)
- Go (Gin, Echo, gRPC)
- Node.js (Express, NestJS, Fastify)
- PHP (Laravel, Weiran 框架)
- Rust (Actix, Rocket, Axum)

✅ **多架构支持**
- 单体架构
- 微服务架构
- SDK/库
- Monorepo

✅ **两阶段工作流**
- **规划模式**: 自动检测项目类型，生成分析计划
- **执行模式**: 按计划逐步生成分析文档

✅ **自动项目检测**
- 自动识别项目类型
- 自动识别架构模式
- 自动识别使用的框架

## 使用方式

### 1. 创建分析计划

```
请为我的项目创建一个代码分析计划
```

技能会自动：
1. 检测项目类型和架构
2. 选择合适的分析模板
3. 生成分阶段的分析计划
4. 请求你的确认

### 2. 执行分析

```
执行项目分析
```

技能会：
1. 加载已有的分析计划
2. 逐步生成分析文档
3. 追踪执行进度
4. 在发现问题时动态调整计划

### 3. 更新计划

```
更新分析计划，添加 [具体要求]
```

技能会：
1. 读取当前计划
2. 应用你的调整
3. 更新进度追踪
4. 请求确认

## 支持的项目类型

### Java 项目

**特征文件**: `pom.xml`, `build.gradle`

**分析重点**:
- 控制器层、服务层、数据访问层
- 依赖注入和 Bean 管理
- 事务管理和并发处理
- 性能优化和缓存策略
- 安全性和测试覆盖

**推荐框架**: Spring Boot, Spring Cloud, Quarkus

### Python 项目

**特征文件**: `requirements.txt`, `setup.py`, `pyproject.toml`

**分析重点**:
- 视图层、业务逻辑层、数据访问层
- 异步处理和任务队列
- 代码风格和类型提示
- 性能优化和缓存
- 安全性和测试覆盖

**推荐框架**: Django, Flask, FastAPI

### Go 项目

**特征文件**: `go.mod`, `go.sum`

**分析重点**:
- 处理器层、业务逻辑层、数据访问层
- Goroutine 和 Channel 使用
- 错误处理和并发安全
- 性能优化
- 安全性和测试覆盖

**推荐框架**: Gin, Echo, gRPC

### PHP 项目

**特征文件**: `composer.json`, `composer.lock`

**分析重点**:
- 控制器层、服务层、数据访问层
- Eloquent ORM 和关系管理
- 事件系统和队列处理
- 权限和策略管理
- 性能优化和缓存

**推荐框架**: Laravel, Weiran 框架

**Weiran 框架特性**:
- 模块化系统分析
- 权限系统分析
- 菜单系统分析
- 服务定义分析
- 钩子系统分析

## 架构模式

### 单体架构

适用于传统的单一代码库应用。

**分析阶段**:
1. 基础结构 (2-3 篇)
2. 领域模块 (每个模块 1 篇)
3. 数据持久化 (2 篇)
4. 外部接入与开放 (2 篇)
5. 跨横切面关注点 (2-3 篇)
6. 总结 (2 篇)

### 微服务架构

适用于分布式系统，多个独立服务。

**分析阶段**:
1. 基础设施 (2-3 篇)
2. 核心服务 (每个服务 1 篇)
3. 网络通信 (2-3 篇)
4. 数据存储层 (2-3 篇)
5. 监控与运维 (2-3 篇)
6. 综合评价 (2-3 篇)

### SDK/库

适用于可复用的代码库。

**分析阶段**:
1. SDK 总览 (2 篇)
2. 核心层级实现 (3-4 篇)
3. 集成引导 (2 篇)
4. 工程质量 (2 篇)

### Monorepo

适用于多个项目在同一仓库。

**分析阶段**:
1. 仓库结构 (2-3 篇)
2. 各项目分析 (每个项目 1-2 篇)
3. 跨项目关注点 (2-3 篇)
4. 综合评价 (2 篇)

## 输出文件

分析计划和文档保存在 `@docs/` 目录下：

```
@docs/
├── plan-analysis.md          # 分析计划主体
├── plan-progress.md          # 进度追踪
├── 01-xxx.md                 # 分析文档 1
├── 02-xxx.md                 # 分析文档 2
└── ...
```

## 文档质量标准

所有分析文档必须包含：

- ✅ 清晰的章节标题
- ✅ 代码示例（包含文件路径）
- ✅ 复杂流程的 Mermaid 图表
- ✅ 设计决策的解释
- ✅ 性能和安全性考量
- ✅ 改进建议

## 工具使用

技能严格遵循以下工具使用原则：

- **文件搜索**: 使用 `Glob` 工具（不使用 bash find）
- **内容检索**: 使用 `Grep` 工具（不使用 bash grep）
- **文件阅读**: 使用 `Read` 工具（不使用 cat/head/tail）
- **并发调用**: 在同一回复中并发调用多个工具以提高性能

## 常见问题

**Q: 如何开始分析？**
A: 在项目根目录运行技能，说"请为我的项目创建一个代码分析计划"。

**Q: 分析需要多长时间？**
A: 取决于项目大小和复杂度。通常一个中等项目需要 2-4 小时。

**Q: 可以中断分析吗？**
A: 可以。进度会被保存，下次可以从中断处继续。

**Q: 如何自定义分析计划？**
A: 在规划阶段提出具体要求，或在执行过程中更新计划。

**Q: 支持哪些编程语言？**
A: Java, Python, Go, Node.js, PHP, Rust 等。

## 技能扩展

要添加新的编程语言支持：

1. 在 `references/project-types/` 目录下创建 `[language].md` 文件
2. 按照现有文件的格式编写分析要点
3. 更新 `scripts/detect_project_type.py` 中的检测逻辑
4. 更新 SKILL.md 中的项目类型列表
