# 多编程语言学习内容自动化生成系统

这是一个多 agent 协作的多编程语言学习内容生成系统，能够系统性地生成结构化的编程语言知识点清单、学习文档和配套单元测试用例。

## 快速开始

**新手？** 先看 [QUICK_START.md](./QUICK_START.md)（5 分钟快速上手）

**想了解工作流？** 见 [WORKFLOW.md](./WORKFLOW.md)（详细执行流程）

**想了解架构？** 见 [ARCHITECTURE.md](./ARCHITECTURE.md)（系统设计）

## 核心功能

### 1. 规划模式（生成学习计划）

当用户要求创建学习计划时，系统会：
- 分析学习需求（语言、领域、水平、目标）
- 设计分阶段学习计划
- 生成 `plan-learning.md` 和 `plan-progress.md`
- 请求用户确认

**输出**：
- `plan-learning.md`：完整的知识点清单
- `plan-progress.md`：进度追踪文件

### 2. 执行模式（生成学习内容）

当用户要求执行学习计划时，系统会：
- 加载已有的计划文件
- 检测增量更新（避免重复生成）
- 分发任务给多个 agent 并行执行
- 生成文档、代码示例、单元测试
- 进行质量检查和验证
- 更新进度追踪文件

**输出**：
- 学习文档：`./learning/docs/`
- 代码示例：`./learning/src/main/[语言]/`
- 单元测试：`./learning/src/test/[语言]/`

## 学习路径

- **完整**（从入门到大师）：8 个阶段，覆盖所有知识点
- **基础**（快速入门）：3 个阶段，掌握核心基础
- **高级**（深入进阶）：3 个阶段，学习高级特性
- **面试**（面试突击）：4 个阶段，重点覆盖高频考点

## 配置

所有配置通过 `language-config.yaml` 统一管理：

```yaml
global_config:
  output_dir: "./learning/"      # 输出目录
  parallel_agents: 3             # 并行 agent 数量
  auto_execution: false          # 是否自动执行
  task_timeout_minutes: 10       # 任务超时时间
  max_retries: 2                 # 最大重试次数
```

**配置优先级**：运行时参数 > 环境变量 > 配置文件默认值

## 多 Agent 协作

系统采用规划-执行两阶段模式：

- **规划阶段**：主 agent 生成整体学习计划
- **执行阶段**：多个子 agent 并行执行具体任务
- **协调层**：主 agent 负责任务分发、结果汇总、质量检查

### 任务分发

- 按知识点独立性分配任务，避免依赖冲突
- 每个 agent 同一时间只处理一个任务
- 复杂知识点分配给经验更丰富的 agent

### 结果汇总

- 所有文档保存到 `{output-dir}/docs/`
- 所有代码保存到 `{output-dir}/src/main/[语言]/`
- 所有测试保存到 `{output-dir}/src/test/[语言]/`

## 质量保证

每个子 agent 完成内容生成后，必须进行：

1. **自校验**：检查文档结构、代码正确性、测试覆盖
2. **代码验证**：编译验证 + 测试执行
3. **主 agent 检查**：格式检查、内容检查、一致性检查

所有代码示例都经过编译和测试验证，可以直接运行。

## 错误处理

### 任务失败

- 第 1 次失败：记录原因，重新分配
- 第 2 次失败：更换 agent 或简化任务
- 第 3 次失败：标记为「阻塞」，记录到 `Adjustment requests`

### 中断恢复

如果执行过程中断，系统会：
1. 读取当前的 `plan-progress.md` 状态
2. 从「待执行」或「进行中」的任务继续
3. 不重复执行已完成的任务

## 常见场景

### 场景 1：我想学习 Java

```
我想学习 Java，从入门到精通。请帮我制定一个完整的学习计划。
```

系统会生成包含 8 个阶段的完整学习计划，涵盖从基础语法到架构设计的所有知识点。

### 场景 2：我需要面试准备

```
我要准备 Python 面试，请生成一个面试突击学习计划。
```

系统会生成重点覆盖高频考点的 4 阶段学习计划。

### 场景 3：我想深入学习某个领域

```
我已经掌握了 Go 的基础，想深入学习并发编程。请生成高级进阶计划。
```

系统会生成针对高级特性和最佳实践的 3 阶段学习计划。

## 性能指标

理想情况下的执行效率：
- **并行度**：3 个 agent 并行执行，效率提升约 2.5 倍
- **吞吐量**：每小时可以生成 6-9 个知识点的完整内容
- **错误率**：内容错误率低于 5%
- **返工率**：需要退回修正的任务比例低于 10%
