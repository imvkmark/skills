# 快速开始指南

## 5 分钟快速上手

### 场景 1：生成 Java 学习计划

```
我想学习 Java，从入门到精通。请帮我制定一个完整的学习计划。
```

**系统会**：
1. 分析你的需求 → 确定目标语言（Java）和学习路径（完整）
2. 生成学习计划 → 创建 `plan-learning.md` 和 `plan-progress.md`
3. 请求确认 → 展示计划摘要，等待你的确认

**你需要做**：
- 查看计划摘要
- 选择「确认执行」或「调整部分内容」

---

### 场景 2：生成学习内容

```
开始执行 Java 学习计划，生成第一阶段的所有文档和测试用例。
```

**系统会**：
1. 加载计划 → 读取 `plan-learning.md` 和 `plan-progress.md`
2. 分发任务 → 将知识点分配给多个 agent 并行执行
3. 生成内容 → 创建文档、代码示例、单元测试
4. 质量检查 → 验证代码可编译、测试可通过
5. 更新进度 → 标记已完成的知识点

**输出位置**：
- 文档：`./learning/docs/`
- 代码：`./learning/src/main/java/`
- 测试：`./learning/src/test/java/`

---

### 场景 3：继续执行下一阶段

```
继续执行 Java 学习计划的第二阶段。
```

**系统会**：
1. 检测进度 → 识别已完成的任务
2. 跳过已完成 → 不重复生成
3. 执行新任务 → 生成第二阶段的内容

## 学习路径说明

- **完整**（从入门到大师）：8 个阶段，覆盖所有知识点
- **基础**（快速入门）：3 个阶段，掌握核心基础
- **高级**（深入进阶）：3 个阶段，学习高级特性
- **面试**（面试突击）：4 个阶段，重点覆盖高频考点

## 常见问题

**Q: 生成的内容在哪里？**
A: 默认在 `./learning/` 目录。可通过 `output-dir` 配置修改。

**Q: 可以修改已生成的计划吗？**
A: 可以。在 `plan-progress.md` 中的 `Adjustment requests` 部分记录修改，系统会在下次执行时应用。

**Q: 如何跳过某个知识点？**
A: 在 `plan-progress.md` 中将该知识点的状态标记为「已跳过」。

**Q: 生成的代码可以直接运行吗？**
A: 可以。所有代码都经过编译和测试验证，可以直接运行。

**Q: 支持增量更新吗？**
A: 支持。系统会检测已生成的文件，只补充缺失部分。
