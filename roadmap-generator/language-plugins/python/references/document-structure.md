### 测试用例编写要求
1. **命名规范**：测试方法名要清晰表达测试意图，使用 `test_xxx` 格式
2. **注释规范**：每个测试方法要有清晰的 docstring 说明测试目的
3. **断言规范**：使用 pytest 提供的断言方法，错误信息要清晰
4. **测试覆盖**：必须覆盖核心功能、边界场景、异常场景
5. **独立性**：每个测试方法独立运行，不依赖其他测试的结果
6. **可重复性**：测试用例可以重复运行，每次结果一致

## 格式规范要求
### Markdown 格式
- 标题层级正确：使用 `#` 一级标题，`##` 二级标题，`###` 三级标题
- 代码块必须指定语言：` ```python `、` ```bash ` 等
- 列表使用统一的符号：无序列表用 `-`，有序列表用数字
- 重要内容用 `**加粗**` 标记，代码变量用 `` `反引号` `` 包裹
- 中文和英文之间要留空格，比如 `Python 3` 而不是 `Python3`

### 代码规范
- 遵循 PEP 8 编码规范
- 使用 4 空格缩进，不要使用 tab
- 变量名、函数名使用小写下划线命名法（snake_case）
- 类名使用大驼峰命名法（PascalCase）
- 常量使用全大写加下划线
- 注释清晰，避免无用注释
- 代码示例要完整，可以直接复制运行

### 内容规范
- 语言通俗易懂，避免过于晦涩的术语
- 示例贴近实际开发，不要用 foo/bar 这种无意义的命名
- 内容准确，避免错误或误导性的表述
- 重点突出，核心内容放在最前面
- 对于容易混淆的概念，要做对比说明
- 特别说明Python 2与Python 3的差异，如果存在
