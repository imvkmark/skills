## 核心组件

### 1. 规划阶段（Planning Phase）

**职责**：
- 分析用户需求
- 设计学习计划
- 生成计划文件

**输入**：
- 目标编程语言
- 学习路径类型
- 学习者水平

**输出**：
- `plan-learning.md`（学习计划主体）
- `plan-progress.md`（进度追踪文件）

**关键决策**：
- 粒度选择（3 级 vs 5 级）
- 阶段划分
- 知识点组织

---

### 2. 执行阶段（Execution Phase）

**职责**：
- 加载计划
- 分发任务
- 执行内容生成
- 质量检查
- 更新进度

**输入**：
- `plan-learning.md`
- `plan-progress.md`
- 语言插件配置

**输出**：
- 学习文档（`docs/`）
- 代码示例（`src/main/[语言]/`）
- 单元测试（`src/test/[语言]/`）
- 更新的进度文件

---

### 3. 协调层（Coordination Layer）

**主 Agent 职责**：
- **任务分发**：将知识点分配给子 agent
- **进度监控**：追踪每个子 agent 的执行状态
- **质量检查**：验证生成的内容
- **冲突解决**：处理 agent 之间的冲突
- **结果汇总**：整合所有子 agent 的输出
- **用户交互**：与用户沟通进度和问题

**关键机制**：
- 并行执行：多个 agent 同时处理不同知识点
- 异步收集：使用 `background_output` 工具收集结果
- 增量更新：检测已生成的文件，避免重复

---

### 4. 语言插件层（Language Plugin Layer）

每种编程语言都有独立的插件，包含：

**配置文件**（`language-config.yaml`）：
- 语言版本
- 构建命令
- 测试框架
- 编码规范
- 目录结构

**参考文档**（`language-plugins/[语言]/references/`）：
- `phase-templates.md`：阶段划分模板
- `document-structure.md`：文档结构规范
- `granularity-guideline.md`：粒度划分指南
- `agent-coordination.md`：多 agent 协作规范

**脚本**（`language-plugins/[语言]/scripts/`）：
- `generate_project_structure.sh`：生成项目结构

## 数据流

### 规划阶段数据流

```
用户需求
    │
    ▼
需求分析 → 确定语言、路径、水平
    │
    ▼
加载语言插件 → 获取阶段模板
    │
    ▼
设计学习计划 → 组织知识点
    │
    ▼
生成计划文件 → plan-learning.md + plan-progress.md
    │
    ▼
展示摘要 → 请求用户确认
    │
    ▼
用户确认 → 更新 Status 为「待执行」
```

### 执行阶段数据流

```
加载计划文件
    │
    ▼
增量更新检测 → 识别待执行任务
    │
    ▼
任务分发 → 分配给多个子 agent
    │
    ├─→ 子Agent1 → 生成文档+代码+测试 → 自校验 → 提交结果
    ├─→ 子Agent2 → 生成文档+代码+测试 → 自校验 → 提交结果
    └─→ 子Agent3 → 生成文档+代码+测试 → 自校验 → 提交结果
    │
    ▼
结果汇总 → 格式检查、内容检查、一致性检查
    │
    ▼
质量检查通过？
    ├─ 否 → 退回修正 → 重新提交
    └─ 是 → 保存文件
    │
    ▼
更新进度文件 → 标记已完成
    │
    ▼
所有任务完成？
    ├─ 否 → 继续下一个知识点
    └─ 是 → 更新 Status 为「已完成」
```

## 多 Agent 协作机制

### 任务分配策略

**原则**：
1. **独立性**：任务之间无依赖关系，可并行执行
2. **粒度适中**：每个任务 10-30 分钟
3. **负载均衡**：任务难度和工作量均衡
4. **专长匹配**：根据 agent 的专长分配任务

**分配流程**：
```
待执行任务列表
    │
    ▼
按难度和领域分类
    │
    ├─ 基础任务 → 分配给所有 agent
    ├─ 中级任务 → 分配给有经验的 agent
    └─ 高级任务 → 分配给专家 agent
    │
    ▼
检查 agent 负载
    │
    ├─ 负载低 → 分配更多任务
    ├─ 负载高 → 等待任务完成
    └─ 质量问题 → 调整到简单任务
    │
    ▼
分发任务 → 并行执行
```

### 结果收集与汇总

**异步收集**：
- 使用 `background_output` 工具异步收集结果
- 不阻塞主 agent 的执行
- 批量处理收集到的结果

**质量检查**：
1. **格式检查**：文件命名、结构、格式
2. **内容检查**：准确性、完整性、正确性
3. **一致性检查**：相关知识点的表述一致
4. **完整性检查**：所有计划内容都已生成

**冲突解决**：
- 内容冲突 → 参考权威资料仲裁
- 资源冲突 → 协调修改顺序
- 进度冲突 → 重新分配任务

## 配置系统

### 全局配置（`language-config.yaml`）

```yaml
global_config:
  output_dir: "./learning/"           # 输出目录
  parallel_agents: 3                  # 并行 agent 数量
  auto_execution: false               # 是否自动执行
  task_timeout_minutes: 10            # 任务超时时间
  max_retries: 2                      # 最大重试次数
```

### 语言特定配置

每种语言的配置包括：
- 版本要求
- 构建命令
- 测试框架
- 编码规范
- 目录结构

### 配置优先级

```
运行时参数 > 环境变量 > language-config.yaml 默认值
```

## 性能指标

### 理想情况下的执行效率

- **并行度**：3 个 agent 并行执行，效率提升约 2.5 倍
- **吞吐量**：每小时可以生成 6-9 个知识点的完整内容
- **错误率**：内容错误率低于 5%
- **返工率**：需要退回修正的任务比例低于 10%

### 优化建议

1. **任务粒度**：适当拆分复杂知识点，提高并行度
2. **质量检查**：前置检查，减少返工
3. **缓存利用**：复用已生成的模板和示例
4. **并行调度**：合理分配任务，避免 agent 空闲

## 扩展性

### 支持新语言

1. 在 `language-config.yaml` 中添加语言配置
2. 创建 `language-plugins/[语言]/` 目录
3. 编写语言特定的参考文档
4. 实现项目结构生成脚本

### 支持新学习路径

1. 在 `language-config.yaml` 中的 `learning_paths` 添加新路径
2. 定义新路径的阶段划分
3. 更新规划阶段的逻辑

### 支持新功能

1. 在协调层添加新的处理步骤
2. 更新子 agent 的执行流程
3. 添加相应的质量检查规则
